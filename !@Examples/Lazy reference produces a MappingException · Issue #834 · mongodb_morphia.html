Given this class, trying to query a document and getting the object User (or whatever object lazy referenced) produces a NullPointerException

@Entity
public class Token implements DatabaseModel {
    @Id
    private ObjectId id;
    private String token;
    @Reference (lazy=true)
    private User user;
    private Date expiration;

//getters, setters, etc.....
}

User is just another POJO containing various fields and embedded classes.

The application runs fine avoiding lazy reference. Anyway, it would be a really nice feature to use....
